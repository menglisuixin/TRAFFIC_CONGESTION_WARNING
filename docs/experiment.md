# 实验与演示说明

本文档用于记录本科毕设演示和论文实验部分可使用的流程、指标和结果分析方式。

## 1. 实验目的

实验目标是验证系统能否完成以下任务：

1. 从道路视频中检测车辆目标。
2. 为车辆分配稳定的 `track_id`。
3. 在 ROI 区域内统计车辆数、流量、密度和占用率。
4. 基于标定或近似比例估算车辆速度。
5. 根据交通指标输出拥堵状态。
6. 生成可播放的可视化视频和结构化 `summary.json`。

## 2. 推荐实验场景

建议至少准备三类视频：

| 场景 | 说明 | 目的 |
| --- | --- | --- |
| 单向道路 | 车辆沿一个方向行驶 | 验证单 ROI 统计和速度估计 |
| 双向道路 | 双向车流同时存在 | 验证 `A_to_B` 和 `B_to_A` 独立统计 |
| 车流变化明显 | 车辆密度有明显变化 | 验证拥堵状态和密度曲线变化 |

如果缺少真实拥堵视频，可以使用普通道路视频进行功能演示，并在论文中说明拥堵规则是可配置的工程规则。

## 3. 实验配置

推荐配置：

```yaml
model:
  weights: yolov5s.pt
  img_size: 640
  conf_thres: 0.25
  classes: [2, 7]

analytics:
  speed_warmup_frames: 3
  speed_history_size: 5
  speed_max_drop_ratio: 0.45
  speed_hold_frames: 8
  max_speed_kmh: 160.0
```

其中 COCO 类别：

- `2`：car
- `7`：truck

## 4. 演示流程

1. 启动服务：

```powershell
conda run -n traffic_warn python service\api.py --host 127.0.0.1 --port 8000
```

2. 打开浏览器：

```text
http://127.0.0.1:8000/
```

3. 上传视频。
4. 在第一帧上绘制 ROI。
5. 如需真实速度，绘制标定四点并填写实际宽度和长度。
6. 点击开始分析。
7. 分析完成后保存以下材料：
   - 输出视频截图。
   - 密度曲线截图。
   - 关键帧统计表截图。
   - `summary.json`。

## 5. 主要指标说明

### 5.1 检测与跟踪指标

- `detections_count`：当前帧 ROI 内目标数量。
- `track_id`：跟踪编号，用于区分不同车辆。
- `label`：类别，当前主要为 `car` 和 `truck`。
- `conf`：检测置信度。

### 5.2 交通流指标

- `vehicle_count`：ROI 内车辆数量。
- `flow_count`：跨越检测线的累计车辆数。
- `density`：车辆数 / ROI 面积。
- `density_per_100k`：每 10 万像素单位的密度，便于展示。
- `occupancy_ratio`：车辆检测框面积总和 / ROI 面积。

### 5.3 速度指标

- `instant_speed_kmh`：瞬时速度。
- `stable_speed_kmh`：稳定速度。
- `speed_kmh`：用于显示和统计的速度。
- `moving_mean_speed_kmh`：当前 ROI 内稳定车辆的平均速度。
- `speed_valid`：是否参与平均速度统计。

`instant_speed_kmh` 表示某一辆车在当前帧根据相邻帧位置变化直接计算出的瞬时速度，它对检测框抖动比较敏感，主要用于排查单帧测速是否异常。`moving_mean_speed_kmh` 表示 ROI 内已经通过稳定性判断的车辆平均速度，经过预热帧过滤、历史速度平滑和异常跳变抑制，更适合作为道路整体速度变化的统计指标。

### 5.4 拥堵状态

- `CLEAR`：畅通。
- `WARNING`：预警。
- `CONGESTED`：拥堵。

拥堵判断主要依据车辆数量、占用率、低速比例和持续帧数。阈值可根据场景调整。

## 6. 结果分析写法示例

可以在论文中这样描述：

```text
实验视频中，系统能够检测道路区域内的车辆，并通过跟踪模块为车辆分配连续 track_id。
在用户绘制的 ROI 区域内，系统统计每帧车辆数、占用率和密度，并结合车辆速度输出交通状态。
从输出视频可以观察到，车辆检测框和 track_id 能够随车辆运动持续显示；从 summary.json 可以进一步提取密度曲线和速度变化趋势。
```

速度部分可以这样描述：

```text
由于目标检测框存在轻微抖动，直接使用相邻帧位移计算速度会产生异常值。
因此系统引入速度预热、历史中位数平滑和瞬时异常抑制策略。
实验结果表明，该策略可以减少车辆刚进入 ROI 或检测框抖动时产生的异常低速，提高平均速度统计的稳定性。
```

## 7. 注意事项

- 视频越清晰、视角越稳定，检测和跟踪效果越好。
- 标定点应位于道路平面，且实际尺寸要尽量准确。
- 双向道路应分开绘制两个 ROI，不建议用一个大 ROI 混合统计。
- 如果浏览器无法播放输出视频，可使用 MJPEG fallback 或检查 FFmpeg/OpenCV 编码环境。
- 若使用官方 `yolov5s.pt`，系统只识别 COCO 中的 `car` 和 `truck`，不会识别自定义类别。

## 8. 建议保存的答辩材料

- 系统首页截图。
- 第一帧 ROI 和标定点截图。
- 分析中或分析后的视频截图。
- 输出视频文件。
- `summary.json` 示例。
- 密度曲线图。
- 关键帧统计表。
- 单向和双向道路对比结果。
