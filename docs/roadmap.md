# 项目路线图

## 1. 已完成内容

### 1.1 检测模块

- 封装 YOLOv5 v7.0 推理流程。
- 支持官方 `yolov5s.pt` 和自训练权重。
- 当前默认检测 COCO 类别 `car` 和 `truck`。
- 输出统一 `Detection` 数据结构。

### 1.2 跟踪模块

- 提供 `BaseTracker` 抽象接口。
- 提供 DeepSORT 兼容实现。
- DeepSORT 依赖不可用时自动使用 IoU Tracker fallback。
- 提供 ByteTrack 风格接口。
- 输出统一 `Track` 数据结构。

### 1.3 分析模块

- ROI 判断。
- 轨迹存储。
- 流量计数。
- 密度估计。
- 占用率估计。
- 速度估计。
- 拥堵状态判断。
- 状态机防抖。

### 1.4 标定与速度

- 支持 Homography 透视变换。
- 支持像素坐标到世界坐标换算。
- 支持真实距离和 km/h 速度计算。
- 已加入速度稳定策略：预热、平滑、突降抑制、抖动保持。

### 1.5 Pipeline

- `TrafficPipeline` 支持视频文件和摄像头流。
- `FrameProcessor` 封装逐帧检测、跟踪、分析和绘制。
- 支持 MP4 输出和 `summary.json` 输出。
- 支持从 `configs/system.yaml` 读取主要配置。

### 1.6 Web 与 API

- FastAPI 服务。
- 视频上传分析接口。
- 第一帧预览接口。
- 结果查询接口。
- 视频和 summary 下载接口。
- Web 前端支持 ROI、双向道路、标定点绘制和结果展示。

### 1.7 测试与工程化

- 已有 pytest 测试。
- 已添加 `.gitignore`。
- 已清理缓存和运行输出目录。
- 项目目录结构适合毕设展示。

## 2. 适合作为毕设展示的功能点

- 上传视频后一键分析。
- 第一帧交互式绘制 ROI。
- 单向道路和双向道路两种模式。
- 检测框、track_id、速度和状态面板可视化。
- 生成可播放视频。
- 生成逐帧 `summary.json`。
- 前端展示密度曲线和关键帧统计表。
- 支持 GPU 推理。

## 3. 后续可优化方向

### 3.1 算法精度

- 使用更强的 YOLOv5 权重或训练更适合道路场景的数据集。
- 正式接入 ByteTrack 或 DeepSORT 依赖，减少 ID 切换。
- 对车辆遮挡和远处小目标进行专项优化。
- 根据真实道路数据标定拥堵阈值。

### 3.2 标定体验

- 保存用户常用 ROI 和标定配置。
- 在前端增加标定点顺序提示。
- 增加标定质量检查，例如提示标定区域是否过小。
- 支持每个方向独立标定。

### 3.3 前端展示

- 增加速度曲线、占用率曲线和流量曲线。
- 支持按 ROI 单独查看统计图。
- 增加分析进度的真实后端进度推送。
- 支持历史分析记录列表。

### 3.4 后端服务

- 增加异步任务队列，避免长视频阻塞请求。
- 增加任务取消和任务进度接口。
- 增加用户上传文件大小限制和异常提示。
- 增加结果自动清理策略。

### 3.5 工程部署

- 提供 Dockerfile。
- 提供一键启动脚本。
- 支持 Linux 服务器部署。
- 支持多摄像头接入。

## 4. 论文展望可写内容

论文最后的展望可以写：

1. 后续可引入更高精度的检测模型，提升远距离车辆和遮挡车辆识别效果。
2. 可引入更稳定的多目标跟踪算法，降低车辆 ID 切换对速度和流量统计的影响。
3. 可结合更多真实道路数据，对拥堵判定阈值进行自适应学习。
4. 可扩展到多摄像头协同分析，实现更大范围的交通状态感知。
5. 可将系统部署到边缘设备或交通管理平台，实现实时预警。
